# BTC 1 小时三分类预测 —— 特征选择与标签设计说明

APS1052 课程项目。本文档解释：(1) 标签阈值的统计依据；(2) 为什么选这 10 个特征；(3) 数据清洗与标签生成规则；(4) 建模阶段怎么用。

## 1. 标签设计与阈值

任务定义：每个整点 t（UTC），用截至 t 的信息预测**未来 1 小时 (t, t+1h]** 内：

- 价格先向上触及 `close_t × (1 + θ)` → **label = +1**
- 价格先向下触及 `close_t × (1 − θ)` → **label = −1**
- 两边都没触及 → **label = 0**

「触及」用 1 分钟 K 线的最高/最低价判断（路径触及，first-touch），盘中摸到就算、不要求收盘价站上去，贴合「一小时以内有没有涨/跌超过 θ」的原意。双向都触及的小时按先触及的一边定标签（triple-barrier 的标准做法）；同一分钟内双向触及的极端情况标 0。

> 决策点为什么取整点：预测时刻理论上可以是任意时间。我们实验过「每整分钟一个决策点」的版本（样本量 ×60），但相邻分钟样本共享 59/60 的前向窗口和几乎全部特征窗口——一次真实行情会产生连续几十个近乎复制品的样本，有效信息并没有增加，反而带来严重的样本相关性问题（评估指标虚高、切分边界泄漏、事件数被夸大 30–60 倍）。因此最终采用整点决策，样本之间前向窗口互不重叠，统计口径干净。

### 阈值的统计依据（全历史 2017-09 → 2026-06，77,128 小时）

| 阈值 θ（路径触及） | 全样本 +1 | 全样本 −1 | 最近两年 +1 | 最近两年 −1 |
|---|---|---|---|---|
| ±5%（最初设想） | 0.23% | 0.32% | 4 次 (0.02%) | 6 次 (0.03%) |
| **±3%（当前采用）** | **0.96%** | **1.25%** | 0.33% | 0.44% |
| ±2% | 2.74% | 3.27% | 0.77% | 0.98% |
| ±1% | 10.96% | 11.86% | 5.36% | 5.92% |

±5% 在最近两年只发生 10 次，完全不可学习，因此放弃。**当前采用 ±3% 作为主任务**：事件含义明确（单小时 3% 是不折不扣的剧烈行情，通常对应清算级联或重大消息），代价是极端的类别不平衡（见第 4 节，建模时必须针对性处理）。脚本阈值是参数（`--threshold`），输出里保留了 `ret_fwd_1h` / `max_up_1h` / `max_dn_1h` 三个原始前向量，随时可以改阈值重新打标签而不必重跑管道。

### 三个阈值成品数据集的分布对比

三个数据集已全部生成（同一套 10 特征、同一清洗规则、同样 50,522 个样本，2020-09 → 2026-06，仅标签不同）：

| | **±3%**（主任务） | **±2%** | **±1%**（对照） |
|---|---|---|---|
| 文件 | `btc_1h_dataset_path_0.03.*` | `btc_1h_dataset_path_0.02.*` | `btc_1h_dataset_path_0.01.*` |
| label −1 | 341（0.67%） | 1,038（2.05%） | 4,611（9.13%） |
| label 0 | 49,956（98.88%） | 48,644（96.28%） | 41,696（82.53%） |
| label +1 | 225（0.45%） | 840（1.66%） | 4,215（8.34%） |
| train 少数类（−1 / +1） | 309 / 203 | 903 / 731 | 3,772 / 3,458 |
| val 少数类（−1 / +1） | 18 / 15 | 75 / 64 | 423 / 407 |
| test 少数类（−1 / +1） | 14 / 7 | 60 / 45 | 416 / 350 |

（train/val/test 为按时间 70/15/15 切分：2020-09→2024-09 / →2025-08 / →2026-06。）

解读：阈值每降 1 个百分点，事件数量大约增加 3–4 倍。**±3% 事件最「有意义」但 test 只有 21 个事件，评估噪声大；±1% 可学习性最好但单个事件的经济含义弱；±2% 是折中**——test 有 105 个事件，勉强够算出稳定一些的 precision/recall。建议报告中三个都跑同一模型做敏感性分析，这比只报一个阈值更有说服力。

注意 BTC 波动率逐年下降（市场机构化），±3% 事件占比从 2021 年的约 3% 降到 2025 年的约 0.4%——这是本任务最主要的非平稳性来源，报告里值得作为发现来讨论。

## 2. 十个特征及选择理由

设计原则：**只用最基础、能一句话解释的指标**，覆盖你要求的几个维度——价格/趋势/波动/成交/情绪，外加衍生品杠杆（BTC 小时级剧烈行情大多由合约市场清算驱动，这一维不可缺）。所有特征在时刻 t 只使用 t 及之前的数据。

| # | 维度 | 特征 | 一句话解释 | 来源 |
|---|---|---|---|---|
| 1 | 价格（短期动量） | `ret_1h` | 过去 1 小时涨跌幅 | Binance 现货 1m K 线 |
| 2 | 趋势（日线方向） | `ret_24h` | 过去 24 小时涨跌幅 | 同上 |
| 3 | 趋势（均线偏离） | `close_vs_sma24` | 现价偏离 24 小时均线的百分比：正 = 价格在均线上方 | 同上 |
| 4 | 情绪（超买超卖） | `rsi_14` | 经典 RSI(14)（小时线）：>70 超买、<30 超卖 | 同上 |
| 5 | 波动 | `rvol_24h` | 过去 24 个小时收益的标准差（已实现波动率） | 同上 |
| 6 | 波动（振幅） | `range_24h` | (24h 最高 − 24h 最低) / 现价 | 同上 |
| 7 | 成交 | `vol_ratio_24h` | 过去 1 小时成交量是 24 小时平均的多少倍：>1 = 放量 | 同上 |
| 8 | 情绪（订单流） | `taker_buy_frac_1h` | 过去 1 小时主动买入量占总成交量比例：>0.5 = 买方主动 | 同上 |
| 9 | 杠杆 | `oi_chg_4h` | 期货未平仓量（OI）过去 4 小时变化率：涨 = 杠杆堆积、市场变脆 | 期货 OI 5 分钟档案 |
| 10 | 情绪（多空拥挤） | `funding_rate` | 最近一期资金费率：极端偏正 = 多头拥挤，易多杀多 | 资金费率 feed（8h） |

**与预测目标的逻辑链**：「下一小时会不会出现 ±3%」首先是波动率问题（#5 #6，波动聚集效应）+ 触发条件问题（#7 放量、#9 杠杆堆积、#10 拥挤度是剧烈行情的前置条件）；「往哪个方向」由动量/趋势（#1 #2 #3）、超买超卖（#4）和买卖压力（#8）提供条件信息。十个特征两两之间信息重叠低，每一个都能在报告里用一句话讲清楚。

### 为什么不选其它数据

- **DVOL 隐含波动率**（日频）：本地数据只到 2025-05，最近 14 个月缺失，测试集没法用。
- **链上指标（MVRV/NUPL/SOPR）、COT 持仓、估值**：日频/周频，反映数周以上的宏观状态，对 1 小时窗口没有分辨率。
- **净主动量日频版**：与 #8 重复且频率低。
- **实时 tick / BBO**：只覆盖最近一小段时间，撑不起 6 年训练样本。
- **Bitstamp K 线**：与 Binance 重复，且无 taker buy volume 字段。
- 上一版里的 30 天成交量 z-score、24h 区间位置、合约多空吃单比：信息与现有特征重叠，且解释成本高，按「基础、可解释」原则替换为 `vol_ratio_24h`、`close_vs_sma24`、`rsi_14`。

## 3. 清洗与防泄漏规则（`scripts/build_dataset.py`）

**时间约定**：全部 UTC。索引 t = 小时 K 线**收盘时刻**，索引 ≤ t 的数值在 t 都已完全可知；标签描述 (t, t+1h]。

1. **1 分钟 K 线**（662MB 合并 CSV）：按 open_time 去重、排序、剔除非正价格；聚合成小时线后**不足 45 根 1 分钟线的小时视为无效**（交易所宕机，152 个小时被剔除），其特征与标签一并作废。
2. **期货 OI**（2,118 个逐日 zip，5 分钟频，2020-09 起）：合并、去重、剔除非正值；`merge_asof(backward, tolerance=2h)` 对齐整点——只取 t 之前最近一条，断档超 2 小时置 NaN 丢弃，不做无限前向填充。
3. **资金费率**（8h 频）：同样 backward as-of，容忍度 9 小时。
4. **标签**：入场价 = t 时刻收盘价；(t, t+1h] 内 1 分钟高低价判断触碰，双向触碰按先后定标签（同一分钟内双触极罕见，标 0）；未来一小时 K 线不完整的样本剔除。
5. **成品**：特征 + 标签合并后 `dropna()`；附带 `close`（回测用，**不进模型**）与三个原始前向量（重打标签用）。

## 4. 成品数据与建模指引

**当前成品**（`data/processed/btc_1h_dataset_path_0.03.parquet` / `.csv`）：

- **50,522 个小时样本**，2020-09-01 → 2026-06-20（起点受 OI 档案限制），无缺失值
- 列 = `close`（非特征）+ 10 特征 + `ret_fwd_1h`/`max_up_1h`/`max_dn_1h`（非特征）+ `label`
- 标签分布：**−1: 341 (0.67%)，0: 49,956 (98.88%)，+1: 225 (0.45%)**

**建模注意事项（±3% 下不平衡非常极端，以下不是可选项而是必须做）**：

1. **切分**：只能按时间顺序切，绝不 shuffle（特征是滚动窗口算的，随机切分 = 信息泄漏 = 虚高分数）。
2. **不平衡处理**：`class_weight="balanced"`（LogisticRegression / RandomForest / LightGBM 均支持），或对 0 类下采样（只在训练集内做）。
3. **评估**：accuracy 无意义（全猜 0 就有 98.9%）。用每类 precision / recall、macro-F1、混淆矩阵；由于 val/test 少数类只有约 15–20 个，**单点指标噪声很大**，建议同时报告 PR 曲线 / PR-AUC，并用 walk-forward（滚动窗口重训）增加测试事件数。
4. **输出用概率**：让模型输出三类概率，再在验证集上调决策阈值（比如 P(±1) 超过某阈值才报警），比直接 argmax 合理——argmax 在 99:1 的先验下几乎永远选 0。
5. **标准化**：线性模型 / 神经网络需要（scaler 只在训练集 fit）；树模型（推荐先跑 LightGBM/RandomForest 基线）可直接用。
6. **报告角度建议**：把 ±3% 主任务与 ±2%、±1% 对照任务放在一起对比——展示阈值 vs 类别平衡 vs 可学习性的权衡，这本身就是很好的课程报告内容。

## 5. 使用方法

```bash
python scripts/build_dataset.py                    # 默认：path 标签, ±3%
python scripts/build_dataset.py --threshold 0.01   # 对照实验
python scripts/build_dataset.py --mode close --threshold 0.03
```

读取示例：

```python
import pandas as pd
ds = pd.read_parquet("data/processed/btc_1h_dataset_path_0.03.parquet")
FEATURES = ["ret_1h", "ret_24h", "close_vs_sma24", "rsi_14", "rvol_24h",
            "range_24h", "vol_ratio_24h", "taker_buy_frac_1h",
            "oi_chg_4h", "funding_rate"]
X, y = ds[FEATURES], ds["label"]
n = len(ds); i70, i85 = int(n*.70), int(n*.85)   # 按时间顺序切分
X_tr, y_tr = X.iloc[:i70], y.iloc[:i70]
X_val, y_val = X.iloc[i70:i85], y.iloc[i70:i85]
X_te, y_te = X.iloc[i85:], y.iloc[i85:]
```
